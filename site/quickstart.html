From zero to fine-tuned Sıfırdan fine-tune'a

Five steps. No notebook. Beş adım. Notebook yok.

Install, pick a template (or bring your own data), validate, train, deploy. The whole loop fits in an afternoon on a single GPU. Kur, bir şablon seç (veya kendi verinizi getir), doğrula, eğit, deploy et. Tüm döngü tek bir GPU'da bir öğleden sonraya sığar.

1

Install Kur

Python 3.10+ on Linux or macOS. The base install gives you the trainer; extras unlock ingestion, GGUF export, and large-corpus deduplication. Linux veya macOS üzerinde Python 3.10+. Temel kurulum trainer'ı verir; ekstralar ingest, GGUF export ve büyük-corpus dedup'ı açar.

terminal 
$ pip install forgelm

# or with extras (pick what you need)
$ pip install 'forgelm[ingestion]'          # PDF/DOCX/EPUB ingest
$ pip install 'forgelm[ingestion-scale]'    # MinHash LSH for >50K rows
$ pip install 'forgelm[export]'             # GGUF export
2

Pick a template — or bring your own YAML Bir şablon seçin — veya kendi YAML'ınızı getirin

Bundled templates ship as YAML for SFT, customer support, code copilot, BYOD domain expert, Turkish medical Q&A, and GRPO math reasoning. Each runs on a single 12 GB GPU out of the box. Yerleşik şablonlar SFT, müşteri destek, kod copilot, BYOD alan uzmanı, Türkçe medikal Q&A ve GRPO matematik akıl yürütmesi için YAML olarak gelir. Her biri tek bir 12 GB GPU'da kutudan çıktığı gibi çalışır.

terminal 
$ forgelm quickstart --list
  customer-support     Multi-turn helpful + safe (SFT + DPO)
  code-assistant       Code-completion fine-tune (SFT + ORPO)
  byod-domain-expert   PDF/DOCX corpus → domain Q&A (SFT)
  medical-qa-tr        Turkish medical Q&A (SFT)
  math-reasoning       Step-by-step reasoning (GRPO)

$ forgelm quickstart customer-support --dry-run
# renders configs/quickstart-customer-support.yaml — open it, edit it, own it.
3

Audit your data first Önce verinizi denetleyin

A leaky train/test split or a credential leaked into your dataset is the most expensive bug you can ship to production. Run the audit before training — it's CPU-only and finishes in seconds on tens of thousands of rows. Sızdıran bir train/test ayrımı veya veri setinize sızmış bir kredensiyel, üretime gönderebileceğiniz en pahalı bug. Eğitimden önce denetimi koşturun — CPU-only ve on binlerce satırda saniyeler içinde biter.

terminal 
# have raw documents? Ingest first.
$ forgelm ingest ./policies/ --recursive --strategy markdown \
    --secrets-mask --pii-mask --output data/policies.jsonl

# then audit (PII / secrets / leakage / quality / language).
$ forgelm audit data/policies.jsonl --output ./audit/

$ jq '.pii_summary, .near_duplicate_pairs, .cross_split_overlap' \
     ./audit/data_audit_report.json
4

Validate, fit-check, then train Doğrula, fit-check, sonra eğit

--dry-run proves the YAML is valid and every referenced file exists. --fit-check tells you whether your VRAM budget will hold before you spend hours discovering it won't. --dry-run YAML'ın geçerli olduğunu ve referans verilen her dosyanın var olduğunu kanıtlar. --fit-check, VRAM bütçenizin tutacağını size saatler harcamadan söyler.

terminal 
$ forgelm --config configs/quickstart-customer-support.yaml --dry-run
✓ config validates
✓ datasets reachable
✓ tokenizer downloadable

$ forgelm --config configs/quickstart-customer-support.yaml --fit-check
FITS  est. peak 11.4 GB / 12 GB available

$ forgelm --config configs/quickstart-customer-support.yaml
# …45 minutes later…
✔ finished, exit 0 — checkpoints/customer-support/
5

Chat, export, deploy Sohbet et, export et, deploy et

Sanity-check the model in the streaming REPL, then quantize for local inference (GGUF), or generate a deployment config for vLLM / Ollama / TGI / HuggingFace Endpoints. Modeli akışlı REPL'de sanity-check edin, sonra yerel inference için kuantize edin (GGUF) veya vLLM / Ollama / TGI / HuggingFace Endpoints için bir deployment config'i üretin.

terminal 
$ forgelm chat ./checkpoints/customer-support
forgelm> how do I cancel my subscription?
…streaming response…

$ forgelm export ./checkpoints/customer-support \
    --output model.gguf --quant q4_k_m

$ forgelm deploy ./checkpoints/customer-support \
    --target ollama --output ./Modelfile

✔ Modelfile written. Run: ollama create my-bot -f ./Modelfile

Prefer a guided wizard? Rehberli sihirbaz mı tercih edersiniz?

forgelm --wizard walks through trainer choice, model, dataset, LoRA rank, evaluation gates — and writes a YAML you can hand-edit afterward. Same code paths, same schema. forgelm --wizard trainer seçimi, model, veri seti, LoRA rank, değerlendirme kapıları arasında gezdirir — ve sonradan elle düzenleyebileceğiniz bir YAML yazar. Aynı kod yolları, aynı şema.
