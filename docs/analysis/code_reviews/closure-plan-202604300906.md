### Faz 31 вҖ” Cross-OS release-tag matrix workflow (Tek PR)

**Closes:** F-test-007 (Major) вҖ” kullanДұcДұ kararДұ: release-tag-only matrix.

**Tasks:**

> **Design note (post-review revisi):** Д°lk taslak `publish.yml`'a `needs: cross-os-tests` ekleyerek cross-workflow dependency kurmaya Г§alДұЕҹДұyordu вҖ” bu GitHub Actions'da **geГ§ersiz** (`needs:` sadece aynДұ workflow iГ§indeki job'lara iЕҹaret edebilir). AyrДұca `pip install -e .` editable install yapДұyordu вҖ” kullanДұcДұnДұn PyPI'dan Г§ekeceДҹi packaged artifact'Дұ (wheel + sdist) test etmiyordu. AЕҹaДҹДұdaki revize tasarДұm her iki sorunu Г§Г¶zer:
> - **Tek workflow:** Cross-OS test job'u + publish job'u **aynДұ `publish.yml`** dosyasДұnda, ikincisi `needs: cross-os-tests` ile birinciye baДҹlДұ.
> - **Packaged artifact testi:** `python -m build` ile wheel + sdist Гјret, sonra wheel'i fresh venv'e `pip install` et, ardДұndan pytest. Editable install yok.

1. `.github/workflows/publish.yml` вҖ” yeniden yapДұlandДұr (tek workflow, iki job):
   ```yaml
   name: Release вҖ” Build, cross-OS validate, publish to PyPI
   on:
     push:
       tags: ['v*']

   jobs:
     build:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
           with: { python-version: '3.11' }
         - name: Build wheel + sdist
           run: |
             python -m pip install --upgrade build
             python -m build  # outputs dist/*.whl and dist/*.tar.gz
         - uses: actions/upload-artifact@v4
           with:
             name: dist
             path: dist/

     cross-os-tests:
       needs: build
       strategy:
         fail-fast: false
         matrix:
           os: [ubuntu-latest, macos-latest, windows-latest]
           python: ['3.10', '3.11', '3.12', '3.13']
       runs-on: ${{ matrix.os }}
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
           with: { python-version: ${{ matrix.python }} }
         - uses: actions/download-artifact@v4
           with: { name: dist, path: dist }
         - name: Install packaged wheel (NOT editable)
           shell: bash   # POSIX shell on Windows runners too
           run: |
             # Pick the .whl built by the upstream build job; install it
             # into a fresh interpreter so the test exercises exactly what
             # PyPI users will pip install.
             python -m pip install --upgrade pip
             python -m pip install dist/*.whl
         - name: Install Linux-only extras (qlora/ingestion/eval)
           if: runner.os == 'Linux'
           # qlora pulls bitsandbytes which only ships Linux wheels.
           # ingestion + eval are cross-platform but bundled here so a
           # single conditional install covers the breadth-test surface.
           # We resolve transitive dependencies from PyPI (no
           # ``--no-index --find-links``) вҖ” the local ``dist/`` only
           # contains the forgelm wheel, not its dependency closure.
           run: python -m pip install 'forgelm[qlora,ingestion,eval]'
         - name: Test (CPU-only, no-network)
           run: pytest tests/ -q --ignore=tests/test_cost_estimation.py
         - name: SBOM generation
           run: python tools/generate_sbom.py > sbom-${{ matrix.os }}-py${{ matrix.python }}.json
         - uses: actions/upload-artifact@v4
           with:
             name: sbom-${{ matrix.os }}-py${{ matrix.python }}
             path: sbom-${{ matrix.os }}-py${{ matrix.python }}.json

     publish:
       needs: cross-os-tests   # blocked until cross-OS green
       runs-on: ubuntu-latest
       environment: pypi   # OIDC trusted publishing
       permissions: { id-token: write, contents: read }
       steps:
         - uses: actions/download-artifact@v4
           with: { name: dist, path: dist }
         - uses: pypa/gh-action-pypi-publish@release/v1
   ```
2. `docs/standards/release.md` вҖ” yeni section "Release prep workflow":

   - `publish` job (gated on `cross-os-tests`): OIDC trusted publishing

3. README'da "OS Independent" claim'i artДұk kanДұtla destekli (3 OS Г— 4 Python green CI matrix gerekГ§esi).
