## 0. Önsöz — Bu plan neyi kapsar, neyi kapsamaz

Bu doküman master review'un 175 bulgusunu (8 Critical + 67 Major + 60 Minor + 40 Nit) **artı** kullanıcı tarafından eklenen 4 yeni kapsamı (Library API, ISO 27001/SOC 2, GDPR full implementation, Article 14 real staging) ★★★★★/5'e ulaşacak şekilde **fazlara dağıtılmış, her fazın sonu PR ile kapanan, risk-yönetilmiş** bir kapanış planına dönüştürür.

### Kullanıcı kararları (2026-04-30)

1. **Tek sürüm:** Bütün bu çalışma **v0.5.5** olarak çıkacak. Ara sürüm yok. Tüm fazlar tamamlanmadan tag atılmaz.
2. **Hedef tarih yok:** Sıralama dependency-driven. Her faz bir önceki faz tamamlanınca başlar.
3. **Faz = PR:** Her faz tek bir PR ile (veya çok-PR'lık coherent bir seri ile) kapanır. Faz içinde task'lar olur.
4. **Library API support:** `forgelm.ForgeTrainer` library API olarak desteklenecek. Önce **detaylı analiz fazı**, sonra **implementation fazı**.
5. **ISO 27001 / SOC 2 Type II:** Pro CLI hedefliyor (Cloud SaaS ayrı ürün, ForgeLM içinde değil). Bu plan'a alignment fazı ekleniyor — analiz + implementation. Açık iş bırakmıyoruz.
6. **GDPR right-to-erasure:** Stub-and-document yerine **gerçek implementation**. Açık iş yok.
7. **Article 14:** Path (a) honesty fix + Path (b) real staging directory + `forgelm approve` subcommand — ikisi birden, bir fazda.
8. **Cross-OS testing:** ubuntu/macos/windows matrix sadece **release-tag** trigger'ında çalışır. Cross-OS yeşil olmadan release publish çalışmaz. Her commit'te değil, sadece release'de.
9. **Pre-commit hooks:** **Opsiyonel** (Soru 8 için kullanıcının önerisini istediği opsiyon; gerekçe §11.7'de).
10. **Final documentation + site sweep:** Tüm fonksiyonel iş bittikten sonra `docs/` (EN+TR) + `docs/usermanuals/{en,tr}/` + `site/*` ayrı bir fazda (Faz 30) v0.5.5 nihai durumuna senkronlanır. **DE/FR/ES/ZH user-manual çevirisi bu fazda yapılmaz** (kullanıcı kararı: "diğer diller için daha sonra çeviri yapacağız").

### Plan disiplini

1. **Tek seferde doğru.** "Şunu da unutmuşuz" yok.
2. **Bisectable PR'lar.** Her faz PR'ı bağımsızca yeşil; revert güvenli.
3. **Sıralı bağımlılıklar açıkça işaretli.** §10 dependency map.
4. **5/5 ne demek?** Her boyut için exit kriteri yazılı (§9).
5. **Açık iş bırakmıyoruz.** GDPR + ISO + Library API gerçek implementation; Article 14 path-b dahil.

### Kapsam dışı (tek satır gerekçeli)

- **DE/FR/ES/ZH user-manual çevirisi** (kullanıcı kararı: deferred — EN+TR yeterli; picker honesty + standards update yapılır; çeviri yok).
- **Cloud SaaS ürün** — ForgeLM içinde değil; ayrı ürün (kullanıcı açıklaması).
- **Yeni roadmap feature'ları** (Phase 13 Pro CLI work, Phase 14 pipeline chains) — bu plan sadece kapanış.

### Faz 31 — Cross-OS release-tag matrix workflow (Tek PR) ✅ TAMAMLANDI — Wave 1 PR

**Closes:** F-test-007 (Major) — kullanıcı kararı: release-tag-only matrix.

**Tasks:**

> **Design note (post-review revisi):** İlk taslak `publish.yml`'a `needs: cross-os-tests` ekleyerek cross-workflow dependency kurmaya çalışıyordu — bu GitHub Actions'da **geçersiz** (`needs:` sadece aynı workflow içindeki job'lara işaret edebilir). Ayrıca `pip install -e .` editable install yapıyordu — kullanıcının PyPI'dan çekeceği packaged artifact'ı (wheel + sdist) test etmiyordu. Aşağıdaki revize tasarım her iki sorunu çözer:
> - **Tek workflow:** Cross-OS test job'u + publish job'u **aynı `publish.yml`** dosyasında, ikincisi `needs: cross-os-tests` ile birinciye bağlı.
> - **Packaged artifact testi:** `python -m build` ile wheel + sdist üret, sonra wheel'i fresh venv'e `pip install` et, ardından pytest. Editable install yok.

1. `.github/workflows/publish.yml` — yeniden yapılandır (tek workflow, iki job):
   ```yaml
   name: Release — Build, cross-OS validate, publish to PyPI
   on:
     push:
       tags: ['v*']

   jobs:
     build:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
           with: { python-version: '3.11' }
         - name: Build wheel + sdist
           run: |
             python -m pip install --upgrade build
             python -m build  # outputs dist/*.whl and dist/*.tar.gz
         - uses: actions/upload-artifact@v4
           with:
             name: dist
             path: dist/

     cross-os-tests:
       needs: build
       strategy:
         fail-fast: false
         matrix:
           os: [ubuntu-latest, macos-latest, windows-latest]
           python: ['3.10', '3.11', '3.12', '3.13']
       runs-on: ${{ matrix.os }}
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-python@v5
           with: { python-version: ${{ matrix.python }} }
         - uses: actions/download-artifact@v4
           with: { name: dist, path: dist }
         - name: Install packaged wheel (NOT editable)
           shell: bash   # POSIX shell on Windows runners too
           run: |
             # Pick the .whl built by the upstream build job; install it
             # into a fresh interpreter so the test exercises exactly what
             # PyPI users will pip install.
             python -m pip install --upgrade pip
             python -m pip install dist/*.whl
         - name: Install Linux-only extras (qlora/ingestion/eval)
           if: runner.os == 'Linux'
           # qlora pulls bitsandbytes which only ships Linux wheels.
           # ingestion + eval are cross-platform but bundled here so a
           # single conditional install covers the breadth-test surface.
           # We resolve transitive dependencies from PyPI (no
           # ``--no-index --find-links``) — the local ``dist/`` only
           # contains the forgelm wheel, not its dependency closure.
           run: python -m pip install 'forgelm[qlora,ingestion,eval]'
         - name: Test (CPU-only, no-network)
           run: pytest tests/ -q --ignore=tests/test_cost_estimation.py
         - name: SBOM generation
           run: python tools/generate_sbom.py > sbom-${{ matrix.os }}-py${{ matrix.python }}.json
         - uses: actions/upload-artifact@v4
           with:
             name: sbom-${{ matrix.os }}-py${{ matrix.python }}
             path: sbom-${{ matrix.os }}-py${{ matrix.python }}.json

     publish:
       needs: cross-os-tests   # blocked until cross-OS green
       runs-on: ubuntu-latest
       environment: pypi   # OIDC trusted publishing
       permissions: { id-token: write, contents: read }
       steps:
         - uses: actions/download-artifact@v4
           with: { name: dist, path: dist }
         - uses: pypa/gh-action-pypi-publish@release/v1
   ```
2. `docs/standards/release.md` — yeni section "Release prep workflow":
   - Tag push trigger
   - `build` job: wheel + sdist
   - `cross-os-tests` job (gated on `build`): 3 OS × 4 Python = 12 kombinasyon, **packaged wheel install**, pytest, SBOM artifact upload
   - `publish` job (gated on `cross-os-tests`): OIDC trusted publishing
   - Hiçbir kombinasyon fail ederse publish çalışmaz
3. README'da "OS Independent" claim'i artık kanıtla destekli (3 OS × 4 Python green CI matrix gerekçesi).
