# Synthetic Data

When real data is scarce or expensive, you can synthesise additional training examples by prompting a stronger model and capturing its responses. ForgeLM's synthetic-data pipeline does this with `forgelm --generate-data`.

## Common use cases

| Use case | What you generate |
|---|---|
| **Distillation** | Teacher (e.g. GPT-4o) answers; student trains on those answers. |
| **Data augmentation** | Paraphrases or variations of your existing data. |
| **Cold-start** | Synthetic examples from a few seed prompts. |
| **Self-instruct** | Prompts and responses both generated. |

## Quick example

```yaml
synthetic:
  enabled: true
  teacher:
    provider: "openai"                  # or "anthropic", "local"
    model: "gpt-4o"
    api_key: "${OPENAI_API_KEY}"
  seed_prompts: "data/seeds.jsonl"      # initial prompts
  output: "data/synthetic.jsonl"
  num_samples: 5000
  temperature: 0.7
  prompt_template: "default"            # or path to custom template
  budget_usd: 50.0                      # cost ceiling

training:
  trainer: "sft"
datasets:
  - path: "data/seeds.jsonl"
  - path: "data/synthetic.jsonl"        # combine seeds + synthetic
```

```shell
$ forgelm --config configs/distill.yaml --generate-data
[2026-04-29 14:01:32] generating from gpt-4o
[2026-04-29 14:01:35] 100/5000 (cost so far: $1.20)
[2026-04-29 14:18:55] 5000/5000 (total cost: $42.30)
✓ wrote 5000 rows to data/synthetic.jsonl
$ forgelm --config configs/distill.yaml          # train normally
```

## Provider support

| Provider | Models | Auth env var |
|---|---|---|
| `openai` | gpt-4o, gpt-4o-mini, o1, o3-mini | `OPENAI_API_KEY` |
| `anthropic` | claude-opus-4, claude-sonnet-4, claude-haiku-4 | `ANTHROPIC_API_KEY` |
| `local` | Any HuggingFace causal LM | none — local inference |
| `vllm` | Any model served via vLLM | `VLLM_BASE_URL` |

For local generation:

```yaml
synthetic:
  teacher:
    provider: "local"
    model: "Qwen/Qwen2.5-72B-Instruct"
    load_in_4bit: true
```

## Cost controls

API-based generation can run up costs quickly. ForgeLM ships hard limits:

```yaml
synthetic:
  budget_usd: 50.0                      # hard ceiling — generation halts here
  max_tokens_per_response: 1024         # cap response length
  rate_limit:
    requests_per_minute: 100            # respect provider rate limits
    burst: 10
```

When the budget is reached, ForgeLM stops with the partial dataset. The audit log records exactly how much was spent.

## Quality controls

Synthetic data is only useful if it's good. ForgeLM's defaults filter:

- **Empty or single-token responses** — the API failed mid-generation; row dropped.
- **Refusals** — pattern-matches "I can't help with that" and similar; row tagged but kept (you might want to keep refusals for safety training).
- **Format failures** — for structured prompts, validates the output fits.

## Citation in the model card

When you train on synthetic data, the auto-generated model card cites the teacher:

```markdown
## Training data

This model was trained on:
- 12,000 rows of human-curated data (data/seeds.jsonl)
- 5,000 rows of synthetic data generated by openai:gpt-4o (2026-04-29)
```

This matters for licensing and commercial use — distilling some teachers requires acknowledging the source.

:::warn
**Check the teacher's terms of service.** Some providers (notably OpenAI for some models) restrict using outputs to train competing models. The model-card citation is a *truth-claim*, not a *licence-grant*. Verify your use is permitted.
:::

## Common pitfalls

:::warn
**Distilling a teacher into a far smaller student.** A 7B student rarely captures a 405B teacher's capabilities. Realistic ratios: 70B → 7B works (with quality loss); 405B → 1.3B doesn't.
:::

:::warn
**Self-instruct without quality filter.** Models generating their own training data tend to drift towards safe, generic outputs. Apply LLM-as-judge ([Judge](#/evaluation/judge)) to filter low-quality synthesised rows.
:::

:::warn
**Forgetting `temperature`.** `temperature=0` produces deterministic responses — the same teacher answer to the same prompt. Use `temperature=0.7` or higher for diversity.
:::
