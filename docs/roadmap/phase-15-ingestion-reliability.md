> **Note:** This file details a single planned phase. See [../roadmap.md](../roadmap.md) for the cross-phase summary.

**Goal:** Close the silent-failure gap in `forgelm ingest` exposed by an end-to-end pilot test against a real-world Turkish-language formal-publication PDF on 2026-05-11. The current pipeline produces output that *looks* successful (no errors, no warnings) but is polluted with multi-line running headers, font-fallback glyphs, bullet-character corruption, page-number leakage, and front-matter noise â€” degrading downstream SFT quality without any operator-facing signal. Phase 15 turns these silent failures into either auto-corrections or loud warnings, **across all five supported formats** (PDF, DOCX, EPUB, TXT, Markdown), and bakes the regression coverage into a fixture matrix.

**Estimated Effort:** Medium (4-6 weeks, Wave 1 + Wave 2)
**Priority:** High â€” gates the credibility of ForgeLM's "config-driven, enterprise-grade" claim on the document ingestion surface. Without Phase 15, real-world Turkish / multi-script corpora produce silently poor data and customers blame the model instead of the pipeline.

> **Context:** Surfaced by two end-to-end pilot runs on 2026-05-11 against a 193-page real-world Turkish-language formal-publication PDF (corpus-agnostic, institutional specifics scrubbed). The first run via the CLI (`--chunk-size 4000`) flagged 74 / 82 chunks below the `--quality-filter` threshold; the second run via `notebooks/ingestion_playground.ipynb` on Colab (`--chunk-size 1024`, both masks on) flagged 161 / 188 chunks. In both runs `forgelm ingest`'s mechanical-success summary reported no issues â€” multi-line running headers, font-fallback glyphs, bullet-character corruption, page-number leakage, recurring institutional-URL pollution, and front-matter chunks all sailed through silently. The same six failure modes generalise across the format Ã— genre Ã— language matrix the pipeline is supposed to handle; the pilots exposed them rather than introduced them. The cross-config replay also empirically verified the determinism guarantee (Turkish character count and U+085F bullet-glyph count were byte-identical across both passes), proving the failures are extraction-layer, not chunking-layer, and that fixes can be locked in with byte-comparison regression fixtures. Phase 15 is the planned remediation programme spanning **CLI surface, library surface, and the user-facing playground notebook** in one coherent v0.6.0 release.

1. [x] **Multi-line PDF edge dedup â€” fix exit-condition trap, not "add an extra check".**
   The current `_strip_repeating_page_lines` in [`forgelm/ingestion.py`](../../forgelm/ingestion.py) is already iterative â€” a `while True:` loop is supposed to peel one edge line at a time and re-check on each pass. The audit (Â§1.1) traced the observed failure to the loop's **exit condition**: pass 1 only inspects `lines[0]` and `lines[-1]`, so when the outermost line varies per page (a per-section title on top, a page number on the bottom) the loop strips zero lines on pass 1 and breaks *before* ever reaching the constant line that sits one row deeper. Future implementers reading just the docstring will assume iterative-peel is enough â€” the comment block above the loop must spell out *why iterative-peel alone fails on variable-outer-line corpora*. Concrete change: replace the pass-1-only `_repeating_edge_lines` with a top-N / bottom-N window check (N = 3 default, exposed as `_PDF_EDGE_WINDOW`) so a line recurring at **any** position within the top-3 / bottom-3 rows across â‰¥ 70 % of pages gets stripped in a single pass. Inline comment (1â€“2 sentences) explains the iteration-trap rationale so this regression cannot return silently. Run a second dedup pass after paragraph packing to catch headers that the chunker re-glued mid-block. Acceptance: on the synthetic Turkish-formal-publication regression fixture, the `"<recurring two-line header>"` pollution drops from â‰¥ 70 % of chunks to â‰¤ 5 %.

2. [x] **Unicode / script sanity layer.**

For each format-detected language (uses existing `langdetect` budget â€” already an `[ingestion]` extra dependency through `langdetect`), computes the ratio of code-points outside the language's expected Unicode blocks. Above a configurable threshold, logs a `WARNING` naming the file and a guess at the cause (font fallback, mojibake, mixed-script noise). Threshold and language hint exposed as `--script-sanity-threshold` and `--language-hint` CLI flags. **Default-threshold calibration is fixture-driven** â€” the audit measured â‰ˆ 7 % corrupt-char ratio on the pilot's front matter and 0 % on clean body chunks, so the placeholder `0.5 %` value in this draft is *intentionally aggressive* and will be re-calibrated against the regression fixture corpus (Task 10) before merge. Final default is set by a precision / recall sweep on the fixture matrix (target: â‰¥ 95 % recall on font-corrupt fixtures, â‰¤ 5 % false-positive rate on clean mixed-script fixtures); likely range `1.0 â€“ 2.0 %`. Acceptance: the front-matter chunks and the custom-glyph-bullet section of the synthetic Turkish-formal-publication fixture each emit one targeted warning *with the calibrated default*, and at least one legitimate mixed-script fixture (e.g. a Turkish corpus with embedded English code samples) emits zero warnings.

#### Wave 2 â€” strong nice-to-have (v0.6.x, ~2 weeks)

1. [x] **`--strip-pattern REGEX` flag â€” operator-controlled regex with ReDoS guard.**
    Operator escape hatch for known-boilerplate that the dedup heuristic misses. Multiple patterns allowed (`--strip-pattern A --strip-pattern B`). Applied after extraction, before chunking. **Operator-supplied regexes must pass the regex-hardening guardrails defined in [`docs/standards/regex.md`](../standards/regex.md)** â€” the same 8-rule discipline (no unbounded greedy quantifiers in sequence, no nested groups with overlapping ranges, bounded repetition) that ForgeLM's own internal regexes obey. Implementation choice: validate the user-supplied pattern at CLI-parse time via the same SonarCloud python:S5852-style structural check ForgeLM applies internally, **and** wrap the match call in a per-pattern timeout (`re.match` with `signal.alarm` or `regex.match(... timeout=N)` if the `regex` library is added as a Wave 2 dependency â€” operator can opt out via `--strip-pattern-no-timeout` if they're certain). Documented in the operator guide with the canonical examples (running headers with variable content, journal DOI lines, watermarks). Acceptance: a pathological `(a+)+b`-class pattern aborts with a clear `EXIT_CONFIG_ERROR (1)` and a message naming the offending pattern, instead of hanging the ingestion process.

2. [x] **`--page-range` PDF flag.**
    Simple `--page-range 12-193` filtering for PDF files. Skips front-matter / back-matter manually when the heuristic in task 13 mis-fires. Validated as `min â‰¤ max`, `min â‰¥ 1`, `max â‰¤ pdf_page_count`. Validation failure **exits with `EXIT_CONFIG_ERROR (1)`** via the CLI dispatcher (matching the existing `_run_ingest_cmd` pattern in [`forgelm/cli/subcommands/_ingest.py`](../../forgelm/cli/subcommands/_ingest.py)) and prints a clear message naming the offending range and the actual page count. Library-level `ingest_path()` raises `ValueError` as before â€” the exit-code translation lives at the CLI boundary per the [error-handling standard](../standards/error-handling.md).

3. [x] **Front-matter heuristic detection.**
    Heuristic that looks at the first N pages of a PDF (N=12 default): if a page has alpha-ratio < 0.45 *and* underscore-ratio > 0.10 *and* contains â‰¥ 5 page-number-pattern matches, the page is tagged as "front-matter candidate" and dropped, with a `WARNING` listing the dropped page indices and the per-page metrics. Opt-out via `--keep-frontmatter`. Symmetric pass on the last N pages catches back-matter (bibliographies, indexes); same threshold.

4. [x] **URL / QR pattern auto-strip option.**
    `--strip-urls` flag: detects URL patterns (http(s)://, www.) per chunk and either replaces with `[URL]` or strips entirely (`--strip-urls=mask|strip|keep`, default `keep`). Useful for exercise-reference QR-noise and academic-paper DOI-noise. Existing PII / secrets mask pipeline integration: applies *after* secrets mask, *before* PII mask, preserving the mask-order discipline. **Interaction with `--all-mask`:** the `--all-mask` shorthand (Phase 12.5) is explicitly *not* extended to include `--strip-urls`. Rationale: PII / secrets are GDPR-grade compliance redactions whose accidental over-application is recoverable (the original text is still in the source PDF); URL stripping is a content-shape decision that operators may want to keep off while still demanding the privacy redactions. The two flag families stay independent and the operator opts into URL handling explicitly. Documented at the CLI help text for both `--all-mask` and `--strip-urls`.

5. [x] **Multi-column PDF detection.**
    Heuristic that estimates column count from pypdf's text-position metadata: per page, cluster x-positions of extracted spans; if two clusters dominate (separation > 30 % of page width), tag the document as 2-column. No fix attempt â€” just a `WARNING` at run start: `[WARN] Detected 2-column layout â€” reading order may be scrambled. Consider --strategy sliding with larger chunk-size, or pre-process with a layout-aware tool.` Better-than-nothing signal; cheaper than a layout-aware extractor.

### Requirements

- **Backward compatibility:** Every Wave 1 / Wave 2 change has an opt-out flag. Existing pipelines that explicitly want the old behaviour (no normalisation, no front-matter skip, no quality pre-signal) get it with a single flag. No silent behaviour change for users with frozen scripts.
- **Determinism preserved â€” at two distinct levels:**
  - **Level 1 (intra-chunk-size byte-identical):** for any fixed `--chunk-size`, `--strategy`, mask flags, and input bytes, the resulting JSONL is byte-identical across runs and across hosts (lex-sorted file walk, no clock dependency, deterministic mask order). This is the property Task 10's fixture-vs-golden byte-compare relies on; the regression suite breaks loudly if any fix introduces nondeterminism here. Documented in [`docs/reference/data_ingestion_architecture.md`](../reference/data_ingestion_architecture.md).
  - **Level 2 (inter-chunk-size count-identical):** at *different* `--chunk-size` values the JSONL bytes naturally diverge (different chunk boundaries), but the **extraction-layer artefact counts** (per-language character counts, font-fallback glyph counts, U+085F bullet counts, redaction counts) remain identical because they are produced by the format extractor *before* chunking. The 2026-05-11 pilot runs empirically verified this: the same source PDF at `--chunk-size 4000` and `--chunk-size 1024` both produced 20,880 Turkish characters and 189 U+085F bullets, byte-for-byte. This invariant justifies the dual-golden fixture strategy in Task 10 â€” the same fixture can carry two byte-identical-per-config goldens *and* a single set of cross-config extraction-count assertions.
   Glyph-normalisation tables are static dicts; quality pre-signal does not change JSONL content, only adds a stderr log line; spine-order EPUB extraction is deterministic. None of these fixes weaken either determinism level.
- **Format coverage symmetry:** Wave 1 must touch all five formats (PDF tasks 1-4, DOCX task 6, EPUB task 7, TXT / MD task 8) **plus the user-facing notebook surface** (task 9). Phase 15 does not ship a "PDF reliability release" that ignores the other four formats, and does not ship a "CLI-only release" that leaves the playground notebook regressed.
- **Notebook surface in-scope:** [`notebooks/ingestion_playground.ipynb`](../../notebooks/ingestion_playground.ipynb) is the canonical "first 5 minutes with ForgeLM ingestion" surface and currently propagates the silent-failure UX even after Tasks 4-5 land at the CLI layer. Task 9 closes that gap. The notebook update PR is gated on **Tasks 4 + 5** landing first (Task 4 produces the ingest-time pre-signal the notebook will surface; Task 5 makes `--quality-filter` default-on so the audit JSON the notebook reads carries the quality data). Task 8 (TXT/MD BOM) is unrelated to the notebook UX and not part of the gating chain. Notebook update ships in the same v0.6.0 release.
- **No new heavy dependencies in Wave 1:** Tasks 1-10 use only `pypdf`, `python-docx`, `ebooklib`, `bs4` (already shipped under `[ingestion]`) and `langdetect` (already shipped). Camelot / pdfplumber / Tesseract are Wave 3 only and gated behind new optional extras.
- **Bilingual docs sweep:** Operator-facing guides updated in EN + TR ([`docs/guides/ingestion.md`](../guides/ingestion.md), [`docs/guides/data_audit.md`](../guides/data_audit.md), [`docs/reference/data_ingestion_architecture.md`](../reference/data_ingestion_architecture.md), and their `-tr.md` mirrors). The bilingual parity guard (`tools/check_bilingual_parity.py --strict`) must pass at PR open. Notebooks are EN-only by project convention and exempt from the bilingual mirror requirement.
- **Honesty in the "Limitations" surfaces:** The architecture reference's existing "Limitations" sections gain a sub-section per still-deferred Wave 3 item (multi-column, OCR, RTL). Users see what does *not* work before they hit it.
- **Test budget:** Wave 1 ships with the 7-fixture regression suite (Task 10) including the dual-golden chunk-size variant on `tr_textbook_4pages.pdf`. Coverage delta target: `forgelm/ingestion.py` line coverage â‰¥ 90 % after Wave 1 lands.
- **No regression on the v0.5.0 contracts:** existing exit codes (0/1/2/3/4/5), the `[REDACTED]` / `[REDACTED-SECRET]` mask vocabulary, the `IngestionResult` schema, and the `--output-format json` envelope are stable across Phase 15. New fields may be added; nothing existing renames or disappears.
